public bool AuthenticateUser(string username, string password)
{
	// Step 1: Validate credentials
	var user = database.GetUserByUsername(username);
	if (user == null || user.Password != password)
	{
		// Step 2: Log failed attempt
		logger.Log($"Failed login attempt for user: {username}");

		// Step 3: Send security alert email
		emailService.Send(user.Email, "Security Alert", "There was a failed login attempt on your account.");
		return false;
	}

	return true;
}

Open/Closed Principle (OCP)

"Software entities should be open for extension, but closed for modification."

‚úÖ Voeg nieuwe functionaliteit toe zonder bestaande code te wijzigen.

‚úÖ Gebruik abstractie (interfaces, inheritance of composition).

‚úÖ Minder risico op het stuk maken van bestaande code.

Liskov Substitution Principle (LSP)

"Derived classes must be substitutable for their base class."

‚úÖ Gedrag uitbreiden, niet veranderen.

‚úÖ Vermijd overrides als dit bestaande verwachtingen verbreekt.

‚úÖ Heroverweeg inheritance (interfaces vs. abstract vs. geen inheritance).

Interface Segregation Principle (ISP)

"Clients should not be forced to depend on interfaces they do not use."

‚úÖ Houd interfaces klein en gefocust. (SRP)

‚úÖ Vermijd grote interfaces die klassen dwingen om irrelevante methoden te implementeren.

Dependency Inversion Principle (DIP)

"Depend on abstractions, not concrete implementations."

‚úÖ Programmeer naar interfaces, niet concrete klassen.

‚úÖ Gebruik Dependency Injection
